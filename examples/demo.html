1. What this page is for

This is a single static HTML page with no backend, no framework, and no build step. Open it in any modern browser and the annotation layer activates automatically. Annotations persist locally in IndexedDB under the database name annotate-js, scoped to this document's path.

The intent of annotate-js is to make AI-assisted document revision practical: a reviewer marks up the page inline, exports a structured JSON brief, and hands the brief plus the original HTML to an AI agent of choice. The agent produces a revised document; the reviewer re-imports the JSON to see which comments were actioned and which still need attention.

2. The review loop

The loop has four steps and one feedback signal:

  1. Mark up — reviewer highlights text and attaches comments inside the browser.
  2. Export — the active session is serialised to a JSON file containing the selected text, surrounding-text context, section heading, comment, and timestamps.
  3. Revise — the JSON and the original HTML are handed to an AI agent. The agent applies each comment and returns a revised HTML document.
  4. Re-import — the reviewer opens the revised HTML, imports the same JSON, and the tool flags each annotation as fresh, moved, or stale based on whether the selected text was found, moved, or no longer found.

2.1 What the flags mean

  • fresh — the exact selected text was found in the new document. The reviewer decides whether the comment is now resolved or still applies.
  • moved — the selected text was found but the surrounding context has shifted. Worth re-reading; the agent likely edited something nearby.
  • stale — the selected text was not found. The agent has almost certainly actioned the comment; verify and dismiss.

3. Why annotation belongs in the AI loop

Existing annotation tools were built for human-to-human review. Comments stay inside the tool, get resolved by another human, and never leave the SaaS. That model breaks down once the resolver is an AI agent: the agent needs a structured brief, not a UI to click through, and the reviewer needs to know which of their points were actioned in the next pass.

The structured JSON export from annotate-js is the contract between the reviewer and the agent. It carries enough context for the agent to locate the change, understand the intent, and apply the edit. The re-anchoring step on import closes the loop so the reviewer is never re-reading the whole document to find out whether their feedback was heard.

4. Privacy posture

All annotations live in the reviewer's browser via IndexedDB. The tool transmits nothing on its own. The only data that ever leaves the device is the JSON file that the reviewer explicitly chooses to export. When sharing the export with an AI agent, treat it as you would any sensitive document — the tool has no opinion on whether the document or the comments are appropriate to share with a third party.
